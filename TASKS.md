# Задачи: Консольное приложение для учёта личных финансов

## Общее описание

Разработать консольное приложение для учёта личных финансов с использованием только стандартной библиотеки Go. Приложение должно работать через командную строку и хранить данные в памяти (на продвинутом этапе — в файле).

---

## Задание 1: Базовая структура и ввод данных (Синтаксис, Структуры)

**Цель:** Закрепить работу со структурами, вводом данных и основными типами.

**Задача:** Создать структуру для представления финансовой операции.

```go
type Transaction struct {
    ID       int
    Amount   float64
    Category string
    Date     string // можно использовать time.Time позже
    Type     string // "income" или "expense"
}
```

**Требования:**
- [ ] Реализовать функцию для создания новой транзакции
- [ ] Реализовать ввод данных с клавиатуры с использованием `fmt.Scan` или `bufio`
- [ ] Валидировать: сумма не может быть отрицательной, тип только `"income"` или `"expense"`

**Пример использования:**
```
Введите тип операции (income/expense): income
Введите сумму: 1500
Введите категорию: salary
```

## Задание 2: Управление коллекцией (Срезы, Мапы)

**Цель:** Научиться работать с коллекциями данных.

**Задача:** Создать менеджер для работы с транзакциями.

```go
type FinanceManager struct {
    transactions []Transaction
    nextID       int
}
```

**Требования:**
- [ ] Метод `AddTransaction(t Transaction)` — добавление новой транзакции
- [ ] Метод `GetAllTransactions() []Transaction` — получение всех транзакций
- [ ] Метод `GetTransactionByID(id int) (*Transaction, error)` — поиск по ID
- [ ] Метод `CalculateBalance() float64` — расчёт общего баланса

**Проверка:**
```go
manager.AddTransaction(Transaction{Amount: 1000, Type: "income"})
manager.AddTransaction(Transaction{Amount: 200, Type: "expense"})
balance := manager.CalculateBalance() // 800
```

**Цель:** Закрепить работу с функциями и условными операторами.

**Задача:** Реализовать систему фильтрации транзакций.

**Требования:**

- [ ] Функция для подсчёта суммы по отфильтрованным транзакциям

## Задание 4: Статистика и отчёты (Интерфейсы)

**Цель:** Понять работу с интерфейсами.

**Задача:** Создать систему генерации отчётов.

```go
type Reporter interface {
    GenerateReport(transactions []Transaction) string
}

type CategoryReport struct{}
type MonthlyReport struct{}
```

**Требования:**
- [ ] Реализовать интерфейс `Reporter` для разных типов отчётов
- [ ] `CategoryReport` — группировка по категориям с суммами
- [ ] `MonthlyReport` — доходы/расходы по месяцам
- [ ] Функция `GenerateReport(r Reporter) string`

## Задание 5: Обработка ошибок

**Цель:** Научиться правильно работать с ошибками.

**Задача:** Добавить обработку ошибок во все критические методы.

**Требования:**
- [ ] Создать кастомные типы ошибок с помощью `errors.New()`
- [ ] Обрабатывать случаи: «транзакция не найдена», «неверный тип операции»
- [ ] Использовать `error` как возвращаемое значение в методах
- [ ] Реализовать graceful degradation при ошибках

## Задание 6: Сохранение в файл (Работа с файлами)

**Цель:** Научиться работать с файловой системой.

**Задача:** Реализовать сохранение и загрузку данных из файла.

**Требования:**
- [ ] Метод `SaveToFile(filename string) error` — сохраняет все транзакции в JSON
- [ ] Метод `LoadFromFile(filename string) error` — загружает транзакции из JSON
- [ ] Использовать `encoding/json` для сериализации
- [ ] Обрабатывать ошибки чтения/записи файлов

**Цель:** Собрать все компоненты в единое приложение.

**Задача:** Создать интерактивный CLI-интерфейс.

**Требования:**

  1. Добавить транзакцию
  2. Показать все транзакции

  4. Показать баланс
  5. Фильтровать транзакции

  7. Сохранить в файл
  8. Загрузить из файла
  9. Выход

- [ ] Цикл работы программы до команды выхода
